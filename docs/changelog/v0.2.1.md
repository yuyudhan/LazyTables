# LazyTables v0.2.1 Detailed Changelog

**Release Date:** October 11, 2025
**Version:** 0.2.1
**Type:** Stable Release
**Previous Version:** 0.1.7

---

## Summary

This release represents a major milestone for LazyTables, transitioning from the beta v0.2.0-beta.1 to a stable release. The core focus has been on architectural improvements, moving to a fully async model that eliminates UI blocking and provides a responsive user experience regardless of database or file I/O performance.

---

## Added Features

### SQL Query Editor Enhancements

#### Vim-style Command Mode
- **Implementation**: Added command mode support in SQL query editor
- **Commands**:
  - `:w` - Save current query to file
  - `:q` - Quit/close query editor
  - `:wq` - Save and quit
- **UX**: Seamless integration with existing vim motions
- **Files Modified**: `src/ui/components/query_editor.rs`

#### SQL Autocomplete System
- **Feature**: Intelligent keyword suggestions and context-aware completions
- **Capabilities**:
  - SQL keyword completion (SELECT, FROM, WHERE, JOIN, etc.)
  - Context-aware suggestions based on cursor position
  - Tab/Enter to accept completions
- **Files Modified**: `src/ui/components/query_editor.rs`

### Navigation Improvements

#### Numbered Pane Navigation
- **Feature**: Direct pane access using number keys 1-6
- **Mapping**:
  - 1 → Connections Pane
  - 2 → Tables Pane
  - 3 → Details Pane
  - 4 → Query Results
  - 5 → SQL Query Editor
  - 6 → SQL Files Browser
- **UX**: Instant pane switching without Tab cycling
- **Files Modified**: `src/app/mod.rs`, `src/ui/mod.rs`

#### Tab Navigation in Table Viewer
- **Keybindings**: Shift+H (previous tab), Shift+L (next tab)
- **Integration**: Works with multiple open tables in query results area
- **Files Modified**: `src/ui/components/table_viewer.rs`

### Visual Feedback Enhancements

#### Animated Loading Indicators
- **Feature**: Non-blocking connection attempts with visual feedback
- **Display**: Timer showing elapsed time during operations
- **Locations**: Connection modal, database queries, file operations
- **Files Modified**: `src/ui/components/connection_modal.rs`, `src/app/mod.rs`

#### Collapsible Table Browser
- **Feature**: Hierarchical organization of database objects
- **Groups**: Schema-based organization, table type grouping (tables, views, materialized views)
- **Navigation**: Expand/collapse with Enter key
- **Files Modified**: `src/ui/components/tables_pane.rs`

### Async Infrastructure

#### Non-blocking Database Connections
- **Architecture**: All database operations run on Tokio async runtime
- **Benefits**: UI remains responsive during connection attempts
- **Timeout**: Configurable timeout protection (default: 30 seconds for connections)
- **Files Modified**: `src/database/connection.rs`, `src/database/postgres.rs`, `src/database/mysql.rs`, `src/database/sqlite.rs`

#### Async File I/O
- **Implementation**: Migrated all file operations to async I/O
- **Timeout**: 5-second default timeout for file operations
- **Scope**: SQL file loading, saving, connection storage, configuration
- **Files Modified**: `src/commands/query.rs`, `src/storage/connection.rs`, `src/config/mod.rs`

## Technical Details

### Async Migration Details

#### Command File Operations
- **Changes**: Migrated all SQL file operations to async I/O
- **Methods**: `load_sql_file()`, `save_sql_file()`, `create_sql_file()`
- **Timeout Protection**: 5-second timeout for all file operations
- **Files Modified**: `src/commands/query.rs`

#### ConnectionStorage Async Conversion
- **Changes**: All connection storage operations now async
- **Methods**: `save_connection()`, `load_connections()`, `delete_connection()`
- **Encryption**: Maintained AES-GCM encryption with async I/O
- **Files Modified**: `src/storage/connection.rs`

#### Application Initialization
- **Changes**: Async initialization sequence
- **Order**: Config loading → Directory creation → Connection loading → UI initialization
- **Error Handling**: Graceful degradation with user feedback
- **Files Modified**: `src/main.rs`, `src/app/mod.rs`

### Error Handling Improvements

- Comprehensive timeout protection for all I/O operations
- Better error messages with user-friendly context
- Graceful degradation when operations fail
- Proper error propagation through async contexts

### Performance Optimizations

- Release profile optimizations: LTO enabled, single codegen unit, strip symbols
- Reduced memory allocations in hot paths
- Efficient async task spawning and management
- Lazy loading for large datasets (foundation for future improvements)
