# Changelog for v0.2.3

**Release Date:** October 14, 2025
**Type:** Maintenance Release (Bug Fixes + Code Refactoring)

---

## 📋 Summary

Version 0.2.3 is a maintenance release focusing on critical bug fixes, major code refactoring to improve maintainability, and several user experience enhancements. The most notable fix addresses a critical issue where number keys were blocked during cell editing.

---

## ✨ Added

### Database Management
- **Database disconnect functionality** (`connections.rs:817-829`)
  - Press 'x' in Connections pane to disconnect from active database
  - Properly cleans up connection state and database objects
  - Shows toast notification confirming disconnection

- **Connection test abort capability** (`app/mod.rs:565-589`)
  - Press Ctrl+C during connection testing to abort ongoing test
  - Aborts background task properly
  - Updates UI with "Test aborted by user" status
  - Shows warning toast notification

### Enhanced Schema Viewing
- **Comprehensive schema metadata display** (`ui/components/table_viewer.rs:1180-1494`)
  - Toggle between Data and Schema views with 't' key
  - **Columns section:** Name, type, nullable status, primary key indicators (🔑)
  - **Indexes section:** Name, type (BTREE/HASH), unique/primary markers, columns, size
  - **Foreign Keys section:** Constraint name, column relationships, ON DELETE/UPDATE actions
  - **Constraints section:** CHECK, UNIQUE, other constraints with definitions
  - **Table Statistics section:** Row count, table/index/total sizes, vacuum/analyze timestamps, owner
  - Scrollable with j/k, gg/G, Ctrl+d/u navigation

### User Experience Improvements
- **Informative empty states** (`ui/mod.rs:863-886`)
  - Query results pane shows helpful guidance when no data is available
  - Clear instructions for getting started

- **Progressive pane enablement** (`app/state.rs:1583-1651`)
  - Panes only enabled when they have relevant data
  - Disabled panes show dimmed borders and informative messages
  - State-aware navigation prevents jumping to disabled panes

## 🐛 Fixed

### Critical Fixes
- **Number key input blocked during cell editing** (`app/handlers/global.rs:293-303`)
  - **Issue:** Pressing number keys 1-6 while editing table cells would trigger pane navigation
  - **Root Cause:** Global key handler processed number keys before table viewer edit mode
  - **Fix:** Added guard condition to check for table edit mode before processing number keys
  - **Impact:** Number keys now correctly enter into edit buffer when editing cells
  - **Files Changed:** `src/app/handlers/global.rs`

### General Fixes
- **Help modal keyboard navigation** (`app/handlers/overlays.rs:108-140`)
  - Fixed event routing for proper help system interaction
  - Help modal now only closes with '?' key, not ESC
  - Proper focus handling for left/right pane navigation

- **SQLite connection URL format** (`app/state.rs:774`)
  - Corrected connection string format: `sqlite://path/to/db.sqlite3`
  - Previously used incorrect format causing connection failures

- **Security improvements in database adapters**
  - Enhanced error handling in PostgreSQL, MySQL, MariaDB, SQLite adapters
  - Better validation of connection parameters
  - Improved error messages for troubleshooting

## 📚 Design Decisions

### Schema Modification Philosophy
**Decision:** Removed table creation and schema editing features

**Rationale:**
- Most users only need to **view** schema information, not modify it
- Power users who need schema modifications should use SQL directly via Query Editor
- Reduces complexity and keeps tool focused on core mission: data viewing and querying
- Schema view remains comprehensive with read-only metadata display

**What Remains:**
- ✅ Comprehensive read-only schema viewing
- ✅ Full SQL query editor for any modifications needed
- ✅ Data viewing and editing capabilities
- ❌ GUI-based table creation (removed)
- ❌ GUI-based schema editing (removed)
