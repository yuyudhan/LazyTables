## Overview

LazyTables v0.1.7 enables full cargo-binstall compatibility, allowing users to install pre-built binaries in seconds rather than waiting for compilation. This release focuses entirely on improving the installation experience while maintaining all existing functionality from v0.1.6.

## 🔧 cargo-binstall Support

### What is cargo-binstall?

cargo-binstall allows you to install Rust binaries without compilation by downloading pre-built binaries from GitHub releases. This dramatically reduces installation time from minutes to seconds.

### Installation Methods

```bash
# NEW: Fast installation with cargo-binstall (recommended)
cargo binstall lazytables

# Standard: cargo install (compiles from source)
cargo install lazytables

# From source
git clone https://github.com/yuyudhan/LazyTables.git
cd LazyTables
cargo install --path .
```

### Supported Platforms

Pre-built binaries are available for:

| Platform | Architecture | Binary Available |
|----------|-------------|------------------|
| macOS | Apple Silicon (M1/M2/M3) | ✅ aarch64-apple-darwin |
| macOS | Intel | ✅ x86_64-apple-darwin |
| Linux | x86_64 | ✅ x86_64-unknown-linux-gnu |
| Linux | ARM64 | ✅ aarch64-unknown-linux-gnu |

## 🔄 Upgrade Path

### From v0.1.6

No breaking changes. Simply update using any installation method:

```bash
# If using cargo-binstall
cargo binstall lazytables --force

# If using cargo install
cargo install lazytables --force

# Configuration and data are preserved
```

### Configuration Compatibility

- All existing connections remain compatible
- SQL files and queries preserved
- No manual migration required
- Backward compatible with v0.1.6 configuration
